## 2024-05-23 - [CRITICAL] Path Traversal in Development Server
**Vulnerability:** The `start.js` development server blindly concatenates user input (`req.url`) to the file system path (`'.' + req.url`) without validation. This allows attackers to access any file on the system (Path Traversal) or sensitive source files in the root directory (Information Disclosure) by requesting URLs like `/.env` or `/server.js`.
**Learning:** Development tools often lack the rigor of production code but can be just as dangerous if they expose secrets or the host machine. In this case, the dev server was listening on a public port (5500) and serving the repository root, mixing frontend assets with backend secrets.
**Prevention:** Always validate and sanitize file paths in custom file servers. Use `path.resolve` to ensure the target path is within the intended directory. Implement an explicit allowlist of safe file extensions or a blocklist of sensitive files/directories, especially when frontend and backend code coexist in the same directory.
