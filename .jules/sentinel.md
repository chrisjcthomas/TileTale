## 2025-12-30 - Fix Missing Cookie Parser & Harden CSRF Cookie
**Vulnerability:** The application was crashing on the authentication callback because `req.cookies` was undefined (missing `cookie-parser` middleware). This rendered the CSRF protection logic (which checks a cookie) completely non-functional and caused a Denial of Service on the auth flow. Additionally, the CSRF cookie lacked the `secure` flag, potentially exposing it over unencrypted connections.
**Learning:** Even if code attempts to implement security checks (like CSRF token verification), missing infrastructure (middleware) can silently (or loudly) disable those checks. Always verify that the underlying mechanisms for security features (like cookie parsing) are present.
**Prevention:** Ensure that dependencies required for security logic (like `cookie-parser` for cookie-based CSRF checks) are installed and configured. Enforce secure defaults (like `secure: true` for cookies) for production environments.
